Details about Nova protocol

Pinhole can be controlled using the Nova protocol.
It responds to these Nova commands:

  • switch_app(appname) // Switch between apps in a group
  • start_app(appname(s)) // Start an app, can be semicolon seperated list
  • start_app_vars(appname;var=val;var=val...) // Start an app with variables
  • stop_app(appname(s)) // Stop an app, can be semicolon seperated list
  • restart_app(appname(s)) // Stop and then restart an app, can be semicolon seperated list
  • stop_group(groupname) // Stop all the apps in a group
  • start_group(groupname) // Start a group
  • stop_all() // Stop all apps
  • start_all() // Start all apps and groups
  • restart() // Reboot the computer
  • shutdown() // Power down the computer
  • log(message) // Log a warning message

The nova protocol contains one command per packet (can be UDP or TCP) and is in the format: 

source:destination:command
Source and destination are in the format: 
site/area/display
Any or all of these values can be left blank in which case any name will match.
The entire field can also be left blank as well to represent a wildcard.
Source addresses are generally only needed if a response is required.
A typical one to one NOVA packet for Monitoring Service might look like: 
::switch_app(fire)
